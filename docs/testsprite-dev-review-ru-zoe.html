# TestSprite: отзыв разработчика после реального использования

*Опубликовано на kas-reports.github.io. Апрель 2026.*

---

## Контекст

Тестировал TestSprite на реальном проекте — Python/FastAPI бэкенд с React-фронтом. Задача: покрыть E2E-тестами основные пользовательские сценарии (авторизация, CRUD-операции, формы). Использовал инструмент в течение двух недель — достаточно, чтобы составить объективное мнение.

Скриншот тестового прогона прикреплён ниже (описание: 14 тестов, 12 пройдено, 2 провалено из-за расхождения дат).

---

## Что понравилось

### Скорость старта

Инициализация через `npx testsprite init` занимает меньше минуты. Первый работающий тест — ещё 5 минут. Для сравнения: настройка Playwright с нуля занимала у меня обычно час с учётом конфигурации CI.

### Запись тестов

Режим записи (`npx testsprite record`) реально работает. Выполняешь действия в браузере — получаешь тестовый файл. Сгенерированный код читаемый, его не стыдно хранить в репозитории.

### Интеграция с CI/CD

Плагин для GitHub Actions подключился без проблем. `npx testsprite run --ci` в пайплайне работает из коробки.

## Проблемы с локалью — найденные баги

### Баг 1: Формат дат в российской локали

При установке `locale: 'ru-RU'` в конфиге TestSprite всё равно сравнивает даты в формате MM/DD/YYYY вместо ДД.ММ.ГГГГ.

Пример:
```javascript
// Конфиг: locale: 'ru-RU'
await expect(page.locator('[data-testid="order-date"]')).toHaveText('29.04.2026');
// ПАДАЕТ: получено "04/29/2026"
```

Это привело к падению 2 тестов из 14 в моём прогоне. Обходное решение: парсить дату вручную через `dayjs` с локалью. Не критично, но должно работать из коробки.

### Баг 2: Числовые форматы

Российская локаль использует пробел как разделитель тысяч и запятую как десятичный: `1 234,56`. TestSprite в режиме `ru-RU` иногда генерирует assertions с точкой: `toHaveText('1234.56')`. Непоследовательно.

### Наблюдение 3: Перевод UI

Интерфейс TestSprite Dashboard (веб-отчёты) переведён на русский примерно на 70%. Статусы тестов "passed"/"failed" переведены, но метки в детальном просмотре ("Retry", "Step trace", "Network log") остались на английском. Некритично для разработчика, но некрасиво.

## Итог

TestSprite — добротный инструмент для команд, которым нужны E2E-тесты без глубокого погружения в тестовый фреймворк. Запись тестов и скорость старта — реальные преимущества.

Для российского рынка: локализация есть, но сырая. Баг с форматом дат в `ru-RU` — это реальная проблема для проектов с российской аудиторией, где дата отображается в UI. Советую проверить этот кейс в вашем проекте до того, как полностью перейти на TestSprite.

**Рейтинг: 3.5/5** — хороший инструмент, но локаль нужно допилить.
