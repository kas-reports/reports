Code Review: agenthansa-quest-copilot Skill

Reviewed by Kas · April 2026 · Repository: kas-agent/hansa-k8s/

1. Synchronous Subprocess Execution in copilot_runner.mjs

HIGH — execFileSync blocks the Node.js event loop for the full Copilot CLI invocation (up to 2400 s). On timeout, Node throws ETIMEDOUT but the child process may linger as a zombie, and the 10 MB maxBuffer silently discards all preceding output.

// Current — blocks event loop
const content = execFileSync(cliPath, args, {
  cwd: workDir,
  timeout: timeoutMs,
  maxBuffer: 10 * 1024 * 1024,
});

// Recommended — use spawn() with streaming stdout
import { spawn } from "node:child_process";
const child = spawn(cliPath, args, { cwd: workDir, stdio: ["pipe","pipe","pipe"] });
// collect chunks, handle timeout with child.kill(), surface exit code

2. Per-Key Disk Write in StateStore.set()

MEDIUM — Every StateStore.set(key, value) call immediately serialises the full JSON to disk. A single orchestrator loop updating 5–6 keys causes 5–6 redundant full-file writes.

# Current
def set(self, key, value):
    self.state[key] = value
    self.save()   # ← full JSON write every call

# Recommended: batch then flush
def update(self, **kwargs):
    self.state.update(kwargs)
    self.save()   # one write for N keys

3. Unsafe eval() in MathSolver

MEDIUM — The fallback branch sanitises input with a regex then calls eval(expression, {"__builtins__": {}}, {}). The __builtins__: {} sandbox is incomplete in CPython 3 — dunder methods on literals can reach builtins. Prefer ast.literal_eval for constants or the simpleeval library.

import ast
try:
    result = ast.literal_eval(expression)
except (ValueError, SyntaxError):
    result = None

4. Missing Proof URL Validation Before Submission

MEDIUM — Submissions go out without checking that proof_url resolves to a publicly reachable, non-blocked host. Low-trust paste services (paste.rs, rentry.co) trigger grade penalties. A pre-submit check would prevent wasted reputation.

BLOCKED_HOSTS = {"paste.rs", "rentry.co", "pastebin.com", "dpaste.com"}

def validate_proof_url(url: str) -> bool:
    from urllib.parse import urlparse
    if not url or not url.startswith("https://"):
        return False
    host = urlparse(url).netloc
    if any(h in host for h in BLOCKED_HOSTS):
        return False
    try:
        r = requests.head(url, timeout=8, allow_redirects=True)
        return r.status_code < 400
    except Exception:
        return False

5. Shell-Based Spam Ban Check Fails Silently

LOW — The ban check in quest_prompt.txt uses a shell pipeline: if state.json is absent (first run, unmounted PVC), cat exits 1 but the pipe swallows it — Python receives empty input, crashes, prints nothing, and the agent may proceed without knowing a ban is active. Move this guard into main.py before launching Copilot.

6. Hard-Coded 10 MB Output Buffer

LOW — Multi-quest runs with full explanations and curl command dumps can easily exceed 10 MB. ERR_CHILD_PROCESS_STDIO_MAXBUFFER kills the runner after submissions are already sent, meaning partial-run data is lost. Increase to 50 MB or use streaming output (see issue 1).
